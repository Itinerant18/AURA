# 09.05 — strategy-service

**Language:** Python + FastAPI (LangChain + Gemini Pro)
**Build order:** #6
**Module mapping:** PRD §4.4

---

## Responsibilities

1. Consume `audit.completed` AND `competitor.report.ready` for a tenant
2. Wait until both arrived for the *same period* before generating strategy
3. Generate 30-day strategy + content calendar via Gemini Pro (LangChain orchestrated)
4. Persist `content_calendars` + child `content_posts` rows (status='draft')
5. Publish `strategy.generated`

---

## The "wait for both" coordination

Strategy needs *both* the audit and the competitor report. Three approaches:

- **A. Stateful join** — service maintains a small `pending_strategy_runs` table; arriving events update flags; when both flags set, kick off generation. **Recommended.**
- B. Polling — periodically check both tables. Wastes cycles, latency unpredictable.
- C. Saga orchestrator — overkill for one join.

Use approach A. Table:

```text
pending_strategy_runs
  tenantId UUID PK
  audit_report_id UUID NULL
  competitor_batch_id UUID NULL  (synthetic — generated by competitor-service per batch)
  ready BOOLEAN
  created_at TIMESTAMPTZ
```

When `ready` flips true, fire the strategy chain.

## LangChain pipeline

`src/chains/strategy_chain.py`:
```text
Input: { tenantId, businessProfile, auditScores, auditInsights, competitorReports, currentDate }
Step 1: PromptTemplate (prompts/strategy_system.txt) → Gemini Pro → StrategyOutline JSON
        Output: { pillars, kpis, platformStrategies, monthlyTheme }
Step 2: PromptTemplate (prompts/calendar_system.txt) → Gemini Pro → 30-day plan
        Output: array of 20-60 slot objects, each:
          { date, platform, contentType, contentPillar, topicHook,
            ctaIdea, hashtagsHint, requiredAssets }
Step 3: Validate with Pydantic
Step 4: Persist:
  - content_calendars row
  - content_posts rows (status='draft', no caption/image yet)
  - outbox_events row for strategy.generated
```

All in one transaction.

## Prompts

`src/prompts/strategy_system.txt` — establishes:
- AURA's voice: practical, data-driven, no fluff
- Output JSON schema (strict)
- Business profile fields, brand voice usage rules
- Platform-specific norms (LinkedIn professional, IG visual, FB community)
- Indian SMB context: Hindi-English hybrid acceptable in captions when brand voice is "friendly" or "playful"

`src/prompts/calendar_system.txt` — establishes:
- 30 days starting from input `currentDate`
- Content mix: 40% image, 25% carousel, 20% reel, 15% story (PRD §4.4.2)
- Posting frequency from audit recommendation
- Distribute pillars evenly across the month
- Each slot uniquely topic-hooked

Prompts versioned (`v1`, `v2`…) and stored alongside code. Changes go through PR review.

---

## Output validation

A `StrategyPlan` Pydantic model defines the strict shape. If Gemini returns invalid JSON:
1. Retry once with a strict reminder appended to the prompt
2. Second failure: write a `strategy_generation_failed` audit log entry, raise → Pub/Sub will retry the original event

## Slot count math

The strategy must produce slots matching the tenant's plan:
- Starter (PRD §7): 20 posts/month
- Growth: 60 posts/month
- Agency: configurable, default 90/month

Read `tenants.plan` to set the budget. Distribute across platforms based on platform strategies in the outline step.
